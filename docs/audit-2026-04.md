## Updated effort estimate

The original gap-analysis estimate was ~30 engineer-days. Based on this audit's findings: **~22–28 engineer-days** for beta-readiness. The codebase is substantially further along than the yellow flags suggested. Most critical paths are wired (feature-gate exists, chat panel is docked-collapsible per design, lesson delivery is 2-at-a-time, observations schema is present, seed data is 80% complete). The main deltas are: (1) UI components missing (progress circles, Smart View, search, messages 3-column, dark mode, observer dashboard), (2) missing chat tools (New Member Form, Professional form), (3) font loading mismatch, and (4) seeded observations. None require architectural changes. With two engineers and clean handoffs across the 7 phases outlined in beta-parity-2026-04.md, the April 15 launch is achievable; one engineer would be tight but doable if items 1–2 (FeatureGate + fonts) are trivial.

## Top 5 concrete items the gap analysis missed

1. **Font loading mismatch** — Layout loads DM Serif Display + Inter, not the Fraunces + Source Serif 4 + Geist trio specified in Design MOC. This is a visual fidelity blocker and needs fixing before Figma comparison work.

2. **Feature-gate exists but not fully wired** — `feature-gate.tsx` component is built (✅); config map and per-family override mutation in Convex schema need verification. The gap analysis listed it as uncertain; it's actually present and just needs audit.

3. **Stewardship phase auto-assign is incomplete** — Logic in `learningBackfill.ts` auto-assigns based on age + role; **advisor override UI (dropdown on member detail page) is missing**. Not listed as a specific gap, but it's a Pillar 1 blocker.

4. **Two seeded observations are missing** — The "pour-over date mismatch" and "ILIT trustee = grantor" observations are supposed to be the "wow" demo item (PRD §19.4). Schema exists; seed data doesn't. This is a narrative/credibility blocker for the Williams demo family.

5. **Chat panel visual spec divergence** — The gap analysis listed it as "verify exact padding vs Figma," but the *design decision* (docked-collapsible per Design MOC.md) supersedes the *PRD spec* (floating with 40/40/40/50 padding per §16.2). This is already correct in code but the gap analysis flagged it as unconfirmed. The PRD is stale here.
